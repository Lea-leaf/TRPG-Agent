# TRPG-Agent

TRPG-Agent 是一个可以在浏览器里游玩的 AI 跑团系统。

它不是单纯让 AI 在聊天框里“编故事”，而是让 AI 扮演 DM，负责描述世界、理解玩家行动、推动剧情；同时由系统记录角色状态、掷骰结果、战斗位置、法术效果、背包奖励和冒险进度。玩家只需要像真正跑团一样说出自己想做什么，剩下的规则结算会在后台完成。

现在的项目已经围绕冒险模组、友方角色、战斗、道具、奖励和地图形成了比较完整的单人跑团体验。

## 你可以怎样游玩

打开页面后，玩家会进入一个聊天式跑团界面。你可以直接用自然语言行动：

```text
我选择法师角色，接受委托出发。
我检查路边的马尸，看有没有埋伏。
我让同伴守住前线，自己退后施放火焰箭。
战斗结束后，我搜索洞穴里的财宝。
我喝一瓶治疗药水，然后继续前进。
```

AI DM 会根据当前剧情和角色状态回应你。如果只是探索和对话，它会继续主持场景；如果涉及检定、战斗、道具、奖励或地图位置，它会调用系统规则来结算，再把结果讲回故事里。

## 冒险模组体验

项目已经接入了《Lost Mine》风格的冒险内容，并整理成可运行的剧情节点。玩家不是在一个空白世界里随便闲聊，而是在一个有地点、线索、遭遇、NPC、奖励和主线方向的冒险中前进。

目前冒险体验包括：

- 从委托出发，沿着道路、伏击、洞穴、城镇等节点推进。
- 场景中包含玩家可见信息和 DM 私密信息，避免关键秘密过早暴露。
- 调查、审问、搜索、战斗胜利等行动会解锁线索或记录事件。
- 剧情会根据已知线索和已完成事件开放下一步。
- 玩家可以回访之前的地点，也可以在合理情况下切换目标。
- 系统会检查 AI DM 是否讲出了当前节点不支持的内容，并在后续回合纠偏。

这让冒险更像“真的在跑一个模组”，而不是每一轮都重新开始即兴创作。

## 友方与队伍感

现在项目已经有了友方角色系统。友方不只是剧情文字里的名字，而是可以进入场景、出现在战斗中、拥有生命值、护甲、武器、法术和行动倾向的单位。

当前已有的友方模板包括：

- 稳重的战士同伴，会保护玩家并尝试稳住前线。
- Sildar 这样的剧情友方，可以作为更强的战士加入冒险。
- 学徒法师，能保持距离并施放基础法术。
- 侍僧治疗者，会优先照顾重伤或倒地的盟友。

友方的存在让单人冒险不再只是“一个人与 AI 对话”，而更接近带着小队探索危险地区的感觉。

## 战斗体验

战斗是当前完成度较高的部分。进入遭遇后，系统会处理先攻、回合顺序、攻击命中、伤害、生命值、移动距离和行动资源。

玩家可以说：

```text
我攻击最近的地精。
我移动到同伴身后，用火球术覆盖那一群敌人。
我对倒地的队友使用治疗药水。
我保留反应，准备施放护盾术。
```

系统会根据角色卡、敌人数据、地图距离和当前回合状态结算结果。前端会显示生命值变化、战斗消息、单位位置和骰子动画，让战斗过程更可见。

项目也采用了新版突袭处理：被突袭者只在开战先攻检定上获得劣势，不会跳过首回合。

## 法术、状态与规则反馈

法术和状态效果已经接入了真实规则结算，不只是文本描述。

已经支持的体验包括：

- 伤害法术，例如火焰箭、火球术、燃烧之手、寒冰匕首。
- 防御法术，例如法师护甲、护盾术、镜影术。
- 控制与标记，例如人类定身术、隐形、目盲、减速、引导箭标记。
- 治疗，例如疗伤术和治疗药水。
- 反应处理，例如被命中时选择是否施放护盾术。

状态会真实影响战斗结果。隐形、麻痹、倒地、魅惑、护盾、镜影、奥术结界等都会进入角色或单位状态，并影响后续判定。

## 道具、奖励与背包

冒险奖励已经可以落到角色卡里。完成节点、发现财宝或解决遭遇后，系统会把奖励加入待领取队列，再由 DM 发放给玩家。

目前支持：

- 经验值奖励。
- 金币和其他货币。
- 剧情财宝。
- 可使用道具。
- 背包面板展示。
- 药水购买和金币扣除。

已有基础药水包括治疗药水、强效治疗药水、隐身药水和活力药水。自己喝药会消耗附赠动作，给别人喂药会消耗动作，并且会检查距离。

## 地图与场景

项目有基础 2D 战术地图。战斗和探索中，单位可以被放置到地图上，系统能计算距离、移动和范围目标。

你可以看到：

- 玩家、敌人和友方的位置。
- 当前地图网格。
- 单位移动后的坐标变化。
- 范围法术可能影响的目标。
- 战斗时的生命值概览。

这让“我离敌人多远”“火球能不能覆盖他们”“我能不能走到队友身边喂药”这些问题有了可计算的依据。

## 界面体验

前端目前提供的是一个跑团主界面，而不是普通聊天页。

主要体验包括：

- 对话流：显示玩家行动、AI DM 回复、战斗事件和工具结算。
- 角色侧栏：查看 HP、AC、属性、资源、法术、武器、状态和背包。
- 战斗面板：观察回合、敌我单位和生命值变化。
- 战术地图：显示单位位置与移动关系。
- 3D D20：掷骰时有立体骰子动画。
- 挂起动作：倒地、反应法术、确认选择等关键节点会等待玩家决定。
- 会话恢复：重新进入已有会话时，可以恢复最近的冒险进度和状态。

## 当前定位

TRPG-Agent 目前更像一个“已经能认真游玩的 AI 单人跑团原型”。它已经具备主线模组、场景推进、规则结算、友方队伍、战斗地图、道具背包和奖励成长这些核心体验。

它还不是完整商业跑团平台。多人联机、更大的模组内容、更复杂的地形与视线、完整职业成长、更多装备和长期 NPC 关系仍在继续完善。

项目的目标很清楚：让 AI 负责想象力，让程序守住规则，让玩家在浏览器里真的经历一场有状态、有骰子、有队友、有奖励的冒险。

## 启动方式

在根目录安装依赖并配置模型服务后，可以用一条命令同时启动前后端：

```powershell
pnpm install
pnpm dev
```

后端模型服务使用兼容 OpenAI 的环境变量：

```powershell
$env:TRPG_LLM_API_KEY="你的 API Key"
$env:TRPG_LLM_BASE_URL="你的模型服务地址"
$env:TRPG_LLM_MODEL="你的模型名"
```
